\doxysubsection{Detailed Description}
Implementation of a weak pointer. \mbox{\hyperlink{classjclib_1_1WeakPointer}{jclib\+::\+Weak\+Pointer}} is a smart pointer that holds a non-\/owning (\char`\"{}weak\char`\"{}) reference to an object that is managed by \mbox{\hyperlink{classjclib_1_1CountedPointer}{jclib\+::\+Counted\+Pointer}}. like std\+::weak\+\_\+ptr, \mbox{\hyperlink{classjclib_1_1WeakPointer}{jclib\+::\+Weak\+Pointer}} models temporary ownership\+: when an object needs to be available only if it exists, and it may be deleted at any time by other code. Like std\+::weak\+\_\+ptr, \mbox{\hyperlink{classjclib_1_1WeakPointer}{jclib\+::\+Weak\+Pointer}} is intended to break circular references in pointers..

This is not the same as std\+::weak\+\_\+ptr, details on that class are at \href{http://en.cppreference.com/w/cpp/memory/weak_ptr}{\texttt{ http\+://en.\+cppreference.\+com/w/cpp/memory/weak\+\_\+ptr}} Major differences\+:
\begin{DoxyItemize}
\item std\+::weak\+\_\+ptr is related to std\+::shared pointer, \mbox{\hyperlink{classjclib_1_1WeakPointer}{jclib\+::\+Weak\+Pointer}} is related to \mbox{\hyperlink{classjclib_1_1CountedPointer}{jclib\+::\+Counted\+Pointer}}
\item std\+::weak\+\_\+pointer may hold references to any object std\+::shared\+\_\+pointer may hold references to. \mbox{\hyperlink{classjclib_1_1WeakPointer}{jclib\+::\+Weak\+Pointer}} may only hold references to objects that publicly inherit from \mbox{\hyperlink{classjclib_1_1WeakPointerTargetBase}{jclib\+::\+Weak\+Pointer\+Target\+Base}}. Usually something that publicly inherits from Weak\+Pointer\+Target$<$\+T$>$
\item \mbox{\hyperlink{classjclib_1_1WeakPointer}{jclib\+::\+Weak\+Pointer}} may hold a nullptr or a value returned from Weak\+Pointer\+Target$<$\+T$>$\+::get\+Weak\+Pointer()
\item std\+::weak\+\_\+ptr must be converted to std\+::shared\+\_\+ptr in order to access the referenced object. \mbox{\hyperlink{classjclib_1_1WeakPointer}{jclib\+::\+Weak\+Pointer}} may be directly dereferenced to a native pointer. That said, unless the pointer is very short lived, converting to a \mbox{\hyperlink{classjclib_1_1CountedPointer}{Counted\+Pointer}} is usually safer. \end{DoxyItemize}
